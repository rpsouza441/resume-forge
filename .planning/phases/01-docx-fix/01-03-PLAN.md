## Prerequisites
1. Run the backend service locally: `cd backend && ./mvnw spring-boot:run`
2. Use the application to generate a DOCX export
3. Download the generated DOCX file

## Visual Verification Steps

### Step 1: Open in Word or LibreOffice
- Open the generated DOCX file in Microsoft Word or LibreOffice Writer
- Verify the document opens without errors

### Step 2: Check Header Section
- [ ] Name appears at top, bold, largest font (16pt/32 half-points)
- [ ] Professional title appears below name
- [ ] Contact information (email, phone, LinkedIn, GitHub) visible
- [ ] Separator line between header and body (not underscores)

### Step 3: Check Font Hierarchy
- [ ] Body text is readable, approximately 10pt
- [ ] Section headings are clearly larger than body text
- [ ] Experience headers (bold) stand out from bullets
- [ ] No text appears too small (< 9pt)

### Step 4: Check Bullet Points
- [ ] Bullets render as actual bullet characters (not dashes or asterisks)
- [ ] Bullets are properly indented
- [ ] Experience highlights use bullet formatting

### Step 5: Check Page Layout
- [ ] Content fits within 2 pages for a standard resume
- [ ] Margins are visible but not excessive
- [ ] No content appears cut off at page edges

### Step 6: Check for Issues
- [ ] No Markdown characters visible (`**`, `###`, `#`)
- [ ] No `{HEADER}` or other template markers
- [ ] No broken text or missing characters
- [ ] Professional overall appearance

## Expected Improvements (vs. Before Fix)
- Margins properly applied (1.8cm/2.0cm)
- Bullets are real OOXML bullets (not simulated)
- Header separator is a clean border line
- Font hierarchy correct (body smaller than headings)
